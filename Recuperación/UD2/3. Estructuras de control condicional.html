3. Estructuras de control condicional

Tipos de estructuras

Como ya hemos estado viendo durante estas semanas, las estructuras condicionales sirven para la toma de decisiones en los algoritmos: si ocurre algo entonces ejecutamos unas sentencias y en caso contrario ejecutamos otras.

Toda toma de decisión tiene una evaluación a realizar para, en función de ella, tomar la decisión. Esa evaluación generalmente compara una variable con otra o contra otro valor, para que en base al resultado de esta comparación, se siga un curso de acción dentro del programa.

Existen tres tipos básicos de estructuras condicionales, en función de las ramas que puedan tener: las simples, las dobles y las múltiples.

3.1. Estructuras condicionales simples (if-then)

A las estructuras condicionales simples se les conoce como "tomas de decisión", y realizan una única acción cuando la expresión a evaluar resulta en un resultado positivo.

Su estructura básica en lenguaje Java es:

if (condición a evaluar){

//sentencias a realizar

}

NOTA: cuando en una condición solamente se indica el nombre de la variable, por ejemplo:

if (variable){

   //sentencias a realizar

}

lo que va a hacer Java es preguntar si el valor de dicha variable es true, es decir, lo supone de tipo booleano. Si quisiéramos escribir la condición contraria, es decir, comprobar si la variable tiene valor false, escribiremos lo siguiente:

if (!variable){

   //sentencias a realizar

}

3.2. Estructuras condicionales dobles (if-then-else)

Las estructuras condicionales dobles permiten elegir entre dos opciones o alternativas posibles, en función del cumplimiento o no de una determinada condición. Por tanto, presentan dos caminos diferentes que puede tomar el flujo de ejecución del programa: si la expresión a evaluar se cumple, el programa se irá por una camino y si no se cumple (else) se va por otro.

Su estructura básica en lenguaje Java es:

if (condición a evaluar){

//sentencias a realizar si se cumple la condición

}else{

//sentencias a realizar si no se cumple la condición

}

3.3. Estructuras condicionales múltiples (if-then-else-if y switch-case)

Las estructuras de comparación múltiples son tomas de decisión que permiten comparar una variable contra distintos posibles resultados. Por lo tanto, en estas estructuras podemos tener no solamente uno o dos caminos, sino tantos como podamos llegar a necesitar en un algoritmo.

Un ejemplo de uso lo tenemos en el programa sobre Generaciones realizado con anterioridad, donde para decidir a cuál pertenecíamos necesitamos implantar la sentencia if-else-if múltiple:

if (anyo_nac>= 1900 && anyo_nac<=1927){
    System.out.println("Eres de la Generación sin bautizar");
} else if (anyo_nac>=1928 && anyo_nac<= 1944) {
    System.out.println("Eres de la Generación Silent");
} else if (anyo_nac>=1945 && anyo_nac<= 1964) {
    System.out.println("Eres de la Generación Baby Boomers");
} else if (anyo_nac>=1965 && anyo_nac<= 1981) {
    System.out.println("Eres de la Generación X");
} else if (anyo_nac>=1982 && anyo_nac<= 1994) {
    System.out.println("Eres de la Generación Millenial");
} else if (anyo_nac>=1995 && anyo_nac <= anyo_actual) {
    System.out.println("Eres de la Generación Centennials");
}else{
    System.out.println("No eres de ninguna generación");
}

Esta estructura en realidad la puedes ver simplemente como una anidación de estructuras condicionales simples y dobles de las que hemos visto anteriormente.

EJERCICIO DE AMPLIACIÓN

Se necesita escribir un programa que pida la altura de una persona.

- Si la altura es menor o igual a 150 cm, mostraremos el mensaje: "Persona de altura baja",

- si la altura está entre 151 y 175 cm, mostraremos el mensaje: "Persona de altura media",

- y si la altura es mayor a 176 cm escriba el mensaje: "Persona alta".

Sentencia switch-case

El switch-case permite comparar una variable, o constante, ante diversos posibles valores. Esta estructura también permite crear varias ramas dentro de un algoritmo con base a distintos posibles valores de la variable, pero ejecutando para cada caso una serie de instrucciones especificas sin necesidad de anidar estructuras más simples (como en el caso anterior if-then-else-if).

Su estructura básica en lenguaje Java es:

switch (variable a evaluar) {
    case value1:
        // secuencia de sentencias.
        break;
    case value2:
       // secuencia de sentencias.
       break;
      .
      .
      .
    case valueN :
      // secuencia de sentencias.
      break;
    default:
      // secuencia de sentencias para cualquier otro valor.
}

Ejemplo:

switch(i) {
  case 0:
    System.out.println("i es cero.");
    break;
  case 1:
    System.out.println("i es uno.");
    break;
  case 2:
    System.out.println("i es dos.");
    break;
  case 3:
    System.out.println("i es tres.");
    break;
  default:
    System.out.println("i no es 1, 2 ni 3");
}

EJERCICIO DE AMPLIACIÓN
Realiza un programa que dado un número entre 1 y 7, escriba su correspondiente día de la semana así: 1- Lunes 2- Martes 3- Miércoles 4- Jueves 5- Viernes 6- Sábado 7- Domingo.

IMPORTANTE: En Java, la sentencia switch sirve para comparar valores estáticos en tiempo de compilación. Es decir, sólo podemos usar switch cuando tenemos valores conocidos. 

Por lo tanto, el ejercicio sobre las Generaciones no se podría resolver estrictamente usando switch. Podríamos forzar el uso de switch así, pero no tiene sentido porque habremos usado igualmente if-else-if:

if (anyo_nac>= 1900 && anyo_nac<=1927){
    valor=1;
} else if (anyo_nac>=1928 && anyo_nac<= 1944) {
    valor=2;
} else if (anyo_nac>=1945 && anyo_nac<= 1964) {
    valor=3;
} else if (anyo_nac>=1965 && anyo_nac<= 1981) {
    valor=4;
} else if (anyo_nac>=1982 && anyo_nac<= 1994) {
    valor=5;
} else if (anyo_nac>=1995 && anyo_nac <= anyo_actual) {
    valor=6;   
}

switch(valor) {
    case 1:
        System.out.println("Eres de la generación sin bautizar");
        break;
    case 2: //código...
        break;
    case 3:
        break;
       .
       .
       .
    default:
}

Lo que sí se puede hacer es agrupar casos de la siguiente manera:

int x = 4;switch(x) {
    case 0:
    case 1:
        System.out.println("Rango entre 0 y 1");
        break;
    case 2:
    case 3:
        System.out.println("Rango entre 2 y 3");
        break;
    case 4:
    case 5:
    case 6:
        System.out.println("Rango entre 4 y 6");
        break;
    default :
        System.out.println("otro rango");
}

Como se puede observar, en este caso el valor a imprimir va a ser "Rango entre 4 y 6", dado que va a ingresar por el case 4, pero al tener el break en el case 6 va a procesar desde case 4 hasta el case 6 y terminará en esa instrucción, dado que existe un break que indica al switch que debe terminar, es decir, salir de él.
